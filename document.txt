Key notes to understand inner calling---

1) if you want to put a listener in queue then
class SendWelcomeEmailListener implements ShouldQueue (call ShouldQueue interface)

2) call queue for Job dispatch() method automatically call laravel for queue if QUEUE_CONNECTION is sync else immediate work and user will wait.

3) For queue ENV file QUEUE_CONNECTION form sync to database

QUEUE_CONNECTION=sync --- Don't queue anything. Run immediately.
QUEUE_CONNECTION=database --- Store jobs in database.Execute later.

4) To call listener add it in EventServiceProvider.php with event name

protected $listen = [
    UserRegistered::class => [
        SendWelcomeEmailListener::class,
    ],
];

5) If queue failed it will store to failed_jobs table
you can run it by "php artisan queue:retry all" / php artisan queue:retry 12(its a job ID)

In live : you will handle it in a separate way like maintain a failed job class
